Three-Axis Recovery Study — System-Specific Final Axis Resolved

A first empirical study testing whether the three axes — Operator, Orientation, and Remainder — are recoverable from 500 completed declarative system-facing sentences from the Perpetual Essays archive through the A-packet process.

1. Research Thesis

This study tests whether completed declarative sentences exhibit a recoverable three-axis completion structure. The sentence is first converted into an A-packet. The A-packet is locked. The resolver then recovers: 

Operator → Orientation → Remainder

The focus is the recovery of the axes. The 54-cell landing is downstream from those recovered axes. 

6 Operators × 3 Orientations × 3 Remainders = 54 possible landings

The central empirical question is: 

Can Operator, Orientation, and Remainder be recovered from each completed declarative sentence through the A-packet process?

2. A-Packet Mechanism

A(S) = <A1, A2, A3, A4, A5, A6>

A1 — settled_saying: the ordinary saying that the sentence leaves settled.

A2 — governing_object: the object, relation, condition, field, role, structure, or demand governed by A1.

A3 — settlement_work: the work performed by A1 upon A2.

A4 — range_scope: the domain, condition, field, or boundary within which A3 holds.

A5 — governing_shape: the structural form left standing after A1 has acted on A2 through A3 within A4.

A6 — excluded_pull: the rival pressure defeated by the packet. A6 remains contrastive and does not govern the landing.

A1 acts on A2 by A3 within A4, leaving A5, while excluding A6.

A(S) = A1 → A2 → A3 @ A4 ⇒ A5 / ¬A6

2A. Mandatory Resolution Protocol: Source-Language A-Packets

Each sentence in the 500-row matrix must be resolved systematically using a large language model. The model must complete A1-A6 for each sentence before resolving Operator, Orientation, Remainder, and Cell.

Mandate: A1, A2, A3, A4, A5, and A6 must be written in the language of the source sentence itself. The A-packet receipt must not be translated into English merely for convenience.

For example: 

If the source sentence is in Irish, A1-A6 must be written in Irish.
If the source sentence is in Welsh, A1-A6 must be written in Welsh.
If the source sentence is in French, A1-A6 must be written in French.
If the source sentence is in Japanese, A1-A6 must be written in Japanese.
If the source sentence is in Chinese, A1-A6 must be written in Chinese.
If the source sentence is in Urdu, A1-A6 must be written in Urdu.
If the source sentence is in Swahili, A1-A6 must be written in Swahili.
If the source sentence is in English, A1-A6 must be written in English.

The large language model may use multilingual reasoning internally, but the visible A-packet fields must remain source-language loyal. If an English gloss, analyst note, or translation aid is needed, it must be placed only in the Notes / strain field and must not replace A1-A6.

The controlled axis labels remain fixed for statistical comparison: 

Operator:
Distinction, Expansion, Relation, Correction, Integration, Closure

Orientation:
Toward, Across, Away

Remainder:
Is, With, Does

Thus, the study separates two tasks: 

1. Source-language A-packet resolution:
   A1-A6 are completed in the language of the sentence.

2. Controlled-axis classification:
   Operator, Orientation, and Remainder are selected from the fixed study vocabulary.

This mandate protects the study from silently converting every sentence into an English paraphrase before resolution. The corpus is multilingual; therefore, the A-packet must show that the sentence can be resolved in its own linguistic surface before being compared through the shared axis system.

3. The Three Axes

Operator

The structural work by which the sentence settles.

Distinction Expansion Relation Correction Integration Closure

Orientation

The direction in which attention resolves.

Toward Across Away

Remainder

The mode of completion left standing.

Is With Does

Operator definitions

Distinction: makes something determinate by fixing, selecting, identifying, marking, bounding, or separating.

Expansion: opens reach, field, availability, continuation, permission, or scope.

Relation: maintains bearing through connection, comparison, dependency, support, recurrence, transfer, or correspondence.

Correction: alters, blocks, refuses, repairs, redirects, or prevents a live prior reading from governing.

Integration: gathers multiple parts, roles, relations, or conditions into one working whole.

Closure: satisfies a live local demand by completing, settling, preserving, validating, returning, or bringing to rest.

Orientation definitions

Toward: attention resolves into a focal target, selected object, role, state, endpoint, or formed unit.

Across: attention holds across a relation, span, comparison, field, dependency, transfer, recurrence, or condition-result structure.

Away: attention resolves by release from, refusal of, preservation against, or movement beyond a rival pressure or prior frame.

Remainder definitions

Is: a standing status, state, identity, condition, availability, placement, role, fixedness, or fact remains.

With: a held bearing, relation, support, comparison, dependency, co-presence, or retained plurality remains.

Does: an enacted outcome, operation, capacity, permission, performance, event, or transformation remains.

4. Pre-Resolution Clarification: Resolution Matrix

Before the 500 sentences are resolved, the resolver must be understood as a matrixed reading of the completed A-packet. The resolver is not guessing from the raw sentence. It is reading structured features from A1-A6.

There are two levels of resolution:

  1. Field-weight level — which A-fields are primary for each axis.
  2. Decision-characteristic level — what features inside those fields justify a specific Operator, Orientation, or Remainder.

Field-weight matrix

Axis resolved Primary fields Supporting fields Final check
Operator A2, A3, A5 A1, A4 A6
Orientation A3, A4, A6 A2, A5 A1
Remainder A5 A2, A3, A4 A6

The packet fields function as follows:

A1 = what is said
A2 = what is governed
A3 = what work is performed
A4 = where the work holds
A5 = what remains standing
A6 = what rival pressure is excluded

The resolver applies this governing logic:

Operator   = the kind of work A3 performs on A2, leaving A5.
Orientation = the direction of attention produced by that work inside A4, with A6 held contrastive.
Remainder   = the mode of the residue named by A5.

It is written before empirical resolution in order to distinguish pre-specified analytic structure from post hoc interpretation. It does not analyse the 500 sentences, does not fill A1-A6, and does not assign Operator, Orientation, Remainder, or Cell values.

The purpose of the study is to test whether the three axes — Operator, Orientation, and Remainder — are recoverable from completed declarative system-facing sentences from the Perpetual Essays archive through the A-packet process. The 54-cell landing is treated as a downstream display of the three recovered axes, not as the primary empirical object of validation. The primary empirical object is axis recoverability.

3. Primary statistical question

The primary statistical question is:

After the 500 sentences are resolved, can the study show that Operator, Orientation, and Remainder were recoverable from the corpus through the A-packet process?

For this study, recovery means that a valid axis value is assigned from the locked A-packet according to the pre-resolution matrix.

The plan distinguishes:

  • A-packet completion: A1, A2, A3, A4, A5, and A6 are all completed.
  • Axis recovery: a valid Operator, Orientation, or Remainder is assigned from the completed A-packet.
  • Full three-axis recovery: Operator, Orientation, and Remainder are all recovered for the same sentence.
  • Cell landing: the downstream 54-cell label is computed from the three recovered axes.
  • Confidence: the resolver's uncertainty signal or self-assessed strength of resolution.
  • Strain: difficulty, ambiguity, competing readings, or instability in the resolution process.

7. Statistical tests for recovery rates

For each binary recovery outcome, the study should report proportions and confidence intervals:

A-packet completion rate
Operator recovery rate
Orientation recovery rate
Remainder recovery rate
full three-axis recovery rate

Recommended methods:

  • Wilson confidence intervals for proportions should be the default interval estimate for completion and recovery rates.
  • Exact binomial confidence intervals may be reported for very high or very low recovery rates or where a conservative interval is desired.
  • One-sample proportion tests may be used against pre-specified success thresholds, but only if thresholds are chosen before analysis.
  • Cochran's Q test should compare paired recovery rates across Operator, Orientation, and Remainder.
  • McNemar tests should be used for pairwise comparisons between axis recovery rates, with multiple-comparison correction.

These tests show whether recovery rates are high in absolute terms and whether one axis is systematically harder to recover than another.

8. Tests for axis distributions

The distributions of Operator, Orientation, and Remainder should be analysed separately before analysing the 54-cell landing.

frequency tables
percentage tables
chi-square goodness-of-fit tests only if expected distributions are pre-specified
entropy or concentration measures

A chi-square goodness-of-fit test should not assume that all categories are equally frequent unless the study explicitly defines uniformity as the expected condition. A system-specific corpus may naturally overrepresent some Operators, Orientations, or Remainders. Unequal distributions do not automatically count against recoverability.

Simple row-level tests are acceptable for initial descriptive analysis and basic recovery-rate summaries. Hierarchical models become necessary when making claims about predictors or stability across repeated forms.

19. Limits of inference

Even strong results would not establish final universality. This study cannot yet establish:

final universality across all sentences
exhaustion of all possible intelligibility structures
semantic saturation of all 54 cells
validity across all genres
validity across all historical periods
validity across oral forms
validity across poetic forms
validity across non-constitutional corpora

The corpus is controlled, English-language, system-facing, and declarative. Therefore, conclusions should be limited to the procedural and empirical scope of this corpus unless later studies extend the method to broader materials.

## Reproducible code

```python
import re, json, math
from pathlib import Path
import pandas as pd
from scipy.stats import chi2_contingency

path = Path('three_axis_recovery_system_specific_final(1).html')
text = path.read_text(encoding='utf-8')

m = re.search(r'const\s+initialRows\s*=\s*', text)
i = m.end()
while text[i].isspace():
    i += 1

start = i
depth = 0
in_str = False
esc = False

for j, ch in enumerate(text[start:], start):
    if in_str:
        if esc:
            esc = False
        elif ch == '\\':
            esc = True
        elif ch == '"':
            in_str = False
    else:
        if ch == '"':
            in_str = True
        elif ch == '[':
            depth += 1
        elif ch == ']':
            depth -= 1
            if depth == 0:
                end = j + 1
                break

rows = json.loads(text[start:end])
df = pd.DataFrame(rows)
df['number'] = df['number'].astype(int)

valid_ops = ['Distinction','Expansion','Relation','Correction','Integration','Closure']
valid_orientations = ['Toward','Across','Away']
valid_remainders = ['Is','With','Does']

df['A_packet_complete'] = df[[f'A{i}' for i in range(1, 7)]].apply(
    lambda row: all(str(x).strip() for x in row),
    axis=1
)

df['Operator_recovered'] = df['operator'].isin(valid_ops)
df['Orientation_recovered'] = df['orientation'].isin(valid_orientations)
df['Remainder_recovered'] = df['remainder'].isin(valid_remainders)
df['Full_axis_recovery'] = (
    df['Operator_recovered']
    & df['Orientation_recovered']
    & df['Remainder_recovered']
)

df['computed_cell_prefix'] = (
    df['remainder'] + ' / ' + df['orientation'] + ' / ' + df['operator']
)

df['cell_valid'] = df.apply(
    lambda row: row['cell'].startswith(row['computed_cell_prefix']),
    axis=1
)

print(len(df))
print(df['cell_valid'].all())
print(df[['A_packet_complete','Operator_recovered','Orientation_recovered','Remainder_recovered','Full_axis_recovery']].sum())
print(pd.crosstab(df['operator'], df['orientation']))
print(pd.crosstab(df['operator'], df['remainder']))
print(pd.crosstab(df['orientation'], df['remainder']))
```
